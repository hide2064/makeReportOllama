### 3.3 各 UI 要素仕様

#### UI-001 ファイルドロップゾーン（Excel/CSV）

| 項目 | 仕様 |
|-----|------|
| 種別 | ドラッグ＆ドロップ + クリックでファイル選択 |
| 対応拡張子 | .xlsx, .xls, .csv |
| ドラッグ中スタイル | 枠線強調・背景色変化 |
| ファイル選択後 | ファイル名表示 + [×] クリアボタン表示 |
| ファイル選択後 | `/api/preview` を自動呼び出し |
| 複数ファイル | 不可（1ファイルのみ） |

#### UI-002 データプレビューテーブル（SCR-001-SUB-01）

| 項目 | 仕様 |
|-----|------|
| 表示条件 | ファイル選択後、`previewData` が存在する場合 |
| コンポーネント | `DataPreview.tsx` |
| 表示内容 | 先頭 5 行のデータテーブル |
| 列バッジ | 必須 6 列それぞれに ✓（存在）or ✗（不足）バッジ |
| 警告表示 | 不足列がある場合に警告メッセージ表示 |
| 行数表示 | 「先頭5行を表示（全XXX行）」の注記 |

**必須列バッジ一覧**

| 列名 | 必須 |
|-----|-----|
| 日付 | ○ |
| 商品名 | ○ |
| 担当者 | ○ |
| 地域 | ○ |
| 数量 | ○ |
| 売上金額 | ○ |
| 利益額 | × (任意) |

#### UI-003 テンプレート選択

| 項目 | 仕様 |
|-----|------|
| 種別 | タブ切替（アップロード / サーバーから選択） |
| サーバーモード | `/api/templates` から取得したファイル名のドロップダウン |
| アップロードモード | PPTX ファイル用ドラッグ＆ドロップエリア |
| 初期値 | サーバーモード・先頭テンプレートを選択 |

#### UI-004 集計期間

| 項目 | 仕様 |
|-----|------|
| 種別 | date 型インプット × 2 |
| バリデーション | 開始 ≤ 終了（違反時は生成ボタン無効化） |
| 任意 | 空欄の場合は全期間を集計 |

#### UI-005 AI モデル選択

| 項目 | 仕様 |
|-----|------|
| 表示条件 | `/api/models` でモデルが取得できた場合のみ表示 |
| Analyst モデル | セレクトボックス（デフォルト: qwen2.5:3b） |
| Writer モデル | セレクトボックス（デフォルト: qwen3:8b） |
| 非表示時 | 環境変数のデフォルトモデルを使用 |

#### UI-006 スライド設定

| 項目 | 仕様 |
|-----|------|
| 商品別表 | チェックボックス（初期: ON） |
| 地域別表 | チェックボックス（初期: OFF） |
| 担当者別表 | チェックボックス（初期: OFF） |
| グラフ | チェックボックス（初期: ON） |
| グラフ種別 | ラジオボタン「棒グラフ / 円グラフ」（初期: 棒グラフ） |
| 依存関係 | グラフ OFF 時はグラフ種別ラジオを無効化 |

#### UI-007 追加指示テキストエリア

| 項目 | 仕様 |
|-----|------|
| 種別 | 複数行テキストエリア |
| 任意 | 空欄可 |
| プレースホルダー | 「例: 前四半期比を強調してください、地域ごとの差異に注目してください」 |
| 上限文字数 | 制限なし（プロンプトに付加） |

#### UI-008 生成ボタン

| 項目 | 仕様 |
|-----|------|
| ラベル | 「レポートを生成する」 |
| 有効条件 | `canSubmit === true` |
| 無効条件 | Excel 未選択 / 必須列不足 / 日付バリデーション失敗 / テンプレート未選択 |
| 押下時 | `App.handleGenerate()` を呼び出し → SCR-004 表示 |

## 7. SCR-005 生成履歴パネル

### 7.1 画面概要

| 項目 | 内容 |
|-----|------|
| コンポーネント | `HistoryPanel.tsx` |
| 表示条件 | 常時表示（メインカードの下） |
| データ取得 | 初回表示時 + `historyKey` 変化時に `/api/history?n=20` を呼び出し |

### 7.2 レイアウト

```
┌─────────────────────────────────────────────────────────────────┐
│  生成履歴                                          [↻ 更新]     │
├─────────────────────────────────────────────────────────────────┤
│  sales_2024Q1.xlsx                                              │
│  2026-03-29 12:34  Analyst: qwen2.5:3b / Writer: qwen3:8b      │
│                                              [↓ DL]            │
├─────────────────────────────────────────────────────────────────┤
│  sales_2024Q2.xlsx                                              │
│  2026-03-28 09:15  Analyst: qwen2.5:3b / Writer: qwen3:8b      │
│                                              [↓ DL]            │
│  ...                                                            │
└─────────────────────────────────────────────────────────────────┘
```

### 7.3 UI 要素仕様

| 要素 | 仕様 |
|-----|------|
| 更新ボタン | クリックで `loadHistory()` を呼び出し再読込 |
| 履歴アイテム | ファイル名 + 生成日時（日本語形式: YYYY年MM月DD日 HH:MM） + モデル情報 |
| DL ボタン | クリックで `/api/history/{job_id}/download` を呼び出しダウンロード |
| 件数 | 最大 20 件表示 |
| 空表示 | 「まだ生成履歴がありません」のメッセージ |

## 8. SCR-006 過去レポート管理パネル（RAG）

### 8.1 画面概要

| 項目 | 内容 |
|-----|------|
| コンポーネント | `ReferenceManager.tsx` |
| 表示条件 | 常時表示（生成履歴パネルの下） |
| データ取得 | 初回表示時に `/api/references` を呼び出し |

### 8.2 レイアウト

```
┌─────────────────────────────────────────────────────────────────┐
│  過去レポート管理 (RAG)              [+ PPTX を登録]            │
│                                                                  │
│  ┌────────────────────────────┬──────────────────────────────┐  │
│  │  登録済みレポート一覧       │  テキストチャンク             │  │
│  │                            │                              │  │
│  │  report_2024Q1.pptx        │  ── スライド 1 ──────────    │  │
│  │  (12 チャンク)  [削除]     │  売上概要...               │  │
│  │                            │                              │  │
│  │  report_2024Q2.pptx ← 選択│  ── スライド 2 ──────────    │  │
│  │  (8 チャンク)   [削除]     │  商品別分析...             │  │
│  │                            │                              │  │
│  └────────────────────────────┴──────────────────────────────┘  │
│                                                                  │
│  ステータスメッセージ: 登録完了: report_2024Q1.pptx (12 chunks)  │
└─────────────────────────────────────────────────────────────────┘
```

### 8.3 UI 要素仕様

| 要素 | 仕様 |
|-----|------|
| 登録ボタン | 「+ PPTX を登録」クリックで `<input type="file">` を起動 |
| 対応ファイル | .pptx のみ |
| アップロード中 | 「アップロード中...」表示（uploading フラグで制御） |
| 登録済みリスト | ファイル名 + チャンク数 + [削除] ボタン |
| 選択 | リストアイテムクリックで右パネルにチャンクを表示 |
| 削除ボタン | クリックで `/api/references/{file_id}` を DELETE 呼び出し |
| チャンクパネル | 選択中ファイルのスライドテキストを `<pre>` タグで表示 |
| ステータス | 操作結果を緑（成功）または赤（エラー）のメッセージで表示 |
| 空表示 | 「過去レポートが登録されていません」のメッセージ |

## 11. バリデーション一覧

| 対象 | チェック内容 | エラー表示場所 |
|-----|-----------|-------------|
| Excel/CSV | 必須列の存在 | データプレビューの列バッジ |
| 日付範囲 | 開始 ≤ 終了 | 日付入力フィールド下 |

| 参照ファイル | PPTX のみ | 参照管理のファイル選択制限 |

## 12. レスポンシブ対応

| 対応 | 仕様 |
|-----|------|
| 対象幅 | 最小 320px〜 |
| レイアウト | Flexbox ベースのシングルカラム |
| テーブル | 横スクロール対応 |
| モバイル最適化 | ボタン・インプットの最小タップ領域 44px |
