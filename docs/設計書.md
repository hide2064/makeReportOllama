# makeReportOllama システム設計書

**バージョン**: 1.0
**作成日**: 2026-03-29
**対象システム**: makeReportOllama — ローカルLLMによる営業報告書自動生成システム

---

## 1. システム概要

### 1.1 目的

Excel / CSV 形式の営業実績データをアップロードするだけで、
ローカル動作の LLM（Ollama）を使用してプロフェッショナルな PowerPoint 報告書を自動生成するシステム。
クラウドサービスや外部 API キー不要で、すべての処理を社内／ローカル環境で完結させる。

### 1.2 主要機能

| 機能 | 説明 |
|------|------|
| 報告書自動生成 | 売上データ → AI分析 → 日本語文章 → PPTX 出力 |
| データプレビュー | アップロード前にデータ内容・列検証を表示 |
| テンプレート管理 | 任意の PPTX テンプレートを使用可能 |
| 生成履歴管理 | 過去の生成物を保存・再ダウンロード |
| RAG 参照管理 | 過去報告書を登録して文章生成品質を向上 |
| ダークモード | UI のライト/ダーク切り替え |

### 1.3 システム特性

- **ローカル完結型**: インターネット接続不要
- **CPU 推論対応**: GPU なしで動作（1回あたり最大約 20 分）
- **同時生成排他制御**: 1件同時生成のみ（多重起動防止）
- **日本語 UI**: 全 UI・出力文章は日本語

## 3. システム処理フロー

### 3.1 報告書生成フロー（メイン処理）

```
[ユーザー] Excel/CSV + テンプレート選択
    ↓
[Frontend] POST /api/generate (FormData)
    ↓
[Backend] バックグラウンドスレッド起動 → {status: "started"} を即時返却
    ↓
[Frontend] GET /api/progress を 2 秒ごとにポーリング
    ↓
━━━━━━━━━━━━━━━━ バックグラウンドスレッド ━━━━━━━━━━━━━━━━
Step 0: Ollama 接続確認 (Analyst + Writer)
Step 1: Excel/CSV 読込 → 集計データ生成
Step 2a: Analyst AI (qwen2.5:3b) → 構造化 JSON
          ※ 失敗時 3 回リトライ → raw_summary にフォールバック
Step 2b: RAG 検索 (ChromaDB cosine 類似度 ≥ 0.3)
Step 2c: Writer AI (qwen3:8b) → 日本語文章 (要約/分析)
Step 3: PPTX 生成 (テンプレート + 表 + グラフ)
Step 4: history.json に記録
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    ↓ done: true
[Frontend] GET /api/download → BLOB URL 作成 → ダウンロードボタン表示
```

### 3.2 RAG 登録フロー

```
[ユーザー] 過去 PPTX アップロード (ReferenceManager)
    ↓
[Backend] POST /api/references/upload
    ↓
python-pptx でスライドテキスト抽出（各スライド＝1チャンク）
    ↓
nomic-embed-text で各チャンクをベクター化 (768次元)
    ↓
ChromaDB に永続保存 (コレクション: "past_reports")
    ↓
[Frontend] 登録済みリスト更新
```

### 3.3 エラーハンドリング方針

| エラー種別 | 対応 |
|-----------|------|
| Ollama 接続失敗 | エラーメッセージ表示（生成中断） |
| Analyst JSON 解析失敗 | 3回リトライ → raw_summary 使用にフォールバック |
| Writer 出力形式不正 | 文章を前半/後半に分割してフォールバック |
| RAG 類似文書なし | RAG スキップして通常生成 |
| 同時生成試行 | 409 Conflict を返却 |
| Excel 列不足 | /api/preview で事前検知・UI で警告表示 |

## 5. セキュリティ設計

### 5.1 CORS 設定
- 許可オリジン: `http://localhost:5173`（開発環境のみ）
- 本番展開時は適切なオリジン制限が必要

### 5.2 ファイルアップロード
- Excel/CSV および PPTX のみ受付（拡張子チェック）
- アップロードファイルは一時ディレクトリに保存し、処理後削除
- `output_path` は履歴 API の一覧レスポンスには含めない（ダウンロード API のみで使用）

### 5.3 ローカル限定
- Ollama・ChromaDB ともに localhost のみで動作
- ネットワーク越しのアクセスは想定外

## 6. 非機能要件

| 項目 | 仕様 |
|------|------|
| 報告書生成時間 | CPU 推論で最大 20 分（qwen3:8b） |
| 同時生成数 | 1 件（排他制御） |
| 履歴保持件数 | 最大 50 件（超過時に古い順から削除） |
| RAG コンテキスト上限 | 1,500 文字 / 最大 5 チャンク |
| 類似度閾値 | 0.3（cosine 類似度） |
| Ollama 接続タイムアウト | 10 秒 |
| 生成タイムアウト | 1,200 秒（20 分） |
| 埋め込みタイムアウト | 60 秒 |
