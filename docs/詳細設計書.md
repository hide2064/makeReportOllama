### 1.3 API ルーター詳細

#### 1.3.1 `routers/report.py` — 報告書生成

**グローバル状態（スレッド共有）**

```python
_status: dict = {
    "step": "",
    "done": False,
    "error": "",
    "output_path": ""
}
_lock: threading.Lock  # ステータス更新の排他制御
_generating: bool = False  # 生成中フラグ
```

**POST /api/generate**

| パラメータ名 | 型 | 必須 | 説明 |
|------------|----|----|------|
| excel_file | File | ○ | Excel/CSV 売上データ |
| template_file | File | △ | PPTX テンプレート（任意） |
| template_name | str | △ | サーバーテンプレート名（任意） |
| slide_product_table | bool | — | 商品別表スライドを含める |
| slide_region_table | bool | — | 地域別表スライドを含める |
| slide_rep_table | bool | — | 担当者別表スライドを含める |
| slide_chart | bool | — | グラフスライドを含める |
| chart_product_type | str | — | "bar" または "pie" |
| analyst_model | str | — | Analyst AI モデル上書き |
| writer_model | str | — | Writer AI モデル上書き |
| date_from | str | — | 集計開始日（YYYY-MM-DD） |
| date_to | str | — | 集計終了日（YYYY-MM-DD） |
| extra_context | str | — | 追加プロンプト指示 |

- 生成中（`_generating == True`）の場合: `HTTP 409 Conflict` を返却
- バックグラウンドスレッドを起動して即時 `{status: "started"}` を返却

**GET /api/progress**

レスポンス:
```json
{
  "step": "Analyst AI が分析中...",
  "done": false,
  "error": "",
  "output_path": ""
}
```

**GET /api/download**

- `_status["output_path"]` が空の場合: `HTTP 400`
- ファイルが存在しない場合: `HTTP 404`
- 存在する場合: `FileResponse` でバイナリ返却

**バックグラウンド処理 `_run_generation()`**

```
1. set_step("Ollama 接続確認...")
   check_ollama(analyst_model)
   check_ollama(writer_model)

2. set_step("Excel / CSV を読み込み中...")
   data = read_and_summarize(excel_path, date_from, date_to)

3. set_step("Analyst AI が分析中...")
   analyst_json = None
   for retry in range(3):
       response = generate(analyst_prompt, analyst_model)
       analyst_json = parse_analyst_json(response)
       if analyst_json: break
   if not analyst_json:
       analyst_json = {"key_facts": [], "concerns": []}  # フォールバック

4. set_step("過去報告書を参照中 (RAG)...")
   rag_context = search_context(data["raw_summary"])

5. set_step("Writer AI が文章を生成中...")
   response = generate(writer_prompt, writer_model)
   summary_text, analysis_text = parse_writer_response(response)

6. set_step("PowerPoint を生成中...")
   output_path = generate_pptx(template_path, output_path, ...)

7. append_history(job_id, filename, output_path, ...)
8. set_done(output_path)
```

#### 1.3.2 `routers/preview.py` — ファイルプレビュー

**POST /api/preview**

処理:
1. アップロードファイルを一時ファイルに保存
2. `pandas.read_excel` または `pandas.read_csv` で読込（Shift-JIS → UTF-8 フォールバック）
3. 先頭 5 行を取得
4. 必須列の存在チェック: `["日付", "商品名", "担当者", "地域", "数量", "売上金額"]`
5. 一時ファイル削除

レスポンス:
```json
{
  "columns": ["日付", "商品名", "担当者", "地域", "数量", "売上金額"],
  "rows": [["2024-01-01", "商品A", "山田太郎", "東京", 10, 100000]],
  "missing_cols": []
}
```

### 1.4 サービス詳細

#### 1.4.1 `services/excel_reader.py`

**`read_and_summarize(file_path, date_from, date_to)`**

| ステップ | 処理 |
|---------|------|

| 前年比計算 | 年別合計から前年比テキストを生成 |
| 利益率計算 | `利益額` 列が存在する場合、月別利益率（%）を計算 |
| ピボット生成 | `quarterly_product_pivot`, `quarterly_region_pivot`, `quarterly_rep_pivot`（上位 8 名） |

戻り値キー一覧:

| キー | 型 | 説明 |
|----|-----|------|
| total_amount | int | 総売上金額 |
| total_qty | int | 総数量 |
| by_product | str | 商品別集計テキスト |
| by_region | str | 地域別集計テキスト |
| by_rep | str | 担当者別集計テキスト |
| period | str | 集計期間文字列 |
| raw_summary | str | Analyst プロンプト用テキスト |
| monthly_totals | dict | {month_str: amount} |
| product_totals | dict | {product_name: amount} 降順ソート済み |
| quarterly_product_pivot | DataFrame | 商品×四半期ピボット |
| quarterly_region_pivot | DataFrame | 地域×四半期ピボット |
| quarterly_rep_pivot | DataFrame | 上位 8 名×四半期ピボット |
| monthly_margin | dict\|None | {month_str: profit_rate%} |

#### 1.4.2 `services/ollama_client.py`

**`check_ollama(model)`**

- `GET {OLLAMA_BASE_URL}/api/tags` を呼び出してモデル存在確認
- 失敗時: `RuntimeError` を送出

**`generate(prompt, model, on_token=None)`**

- `POST {OLLAMA_BASE_URL}/api/generate` にストリーミングリクエスト
- 30 トークンごとに `on_token(token_text)` を呼び出し（進捗更新用）
- qwen3 の `<think>...</think>` ブロックを正規表現で除去
- 戻り値: 生成テキスト全文

**Analyst プロンプト仕様**

```
システム指示: JSON のみ出力。余分なテキスト禁止。
出力 JSON スキーマ:
{
  "period": string,
  "total_sales": number,
  "total_qty": number,
  "top_products": [{"name": string, "amount": number}],
  "bottom_products": [...],
  "top_regions": [...],
  "bottom_regions": [...],
  "top_reps": [...],
  "key_facts": [string],       # 箇条書き事実
  "concerns": [string],         # 懸念事項
  "yoy_change": {year: percent_str}
}
```

**Writer プロンプト仕様**

- 文体: ビジネス日本語、箇条書き、数値根拠付き
- 出力フォーマット: `---SUMMARY---` と `---ANALYSIS---` セクション
- 各セクション: 150〜200 文字
- RAG コンテキスト: `[参考: 過去報告書の記述]` として付加
- ユーザー追加指示: `[追加指示]` として付加

**`parse_analyst_json(response)`**

- マークダウンコードブロック（` ```json ... ``` `）の除去
- `json.loads()` でパース
- 失敗時: 空辞書 `{}` を返却

**`parse_writer_response(response)`**

- `---SUMMARY---` と `---ANALYSIS---` で分割
- どちらか欠けている場合: テキストを前半/後半に分割してフォールバック

#### 1.4.3 `services/pptx_generator.py`

**`generate_pptx(template_path, output_path, summary_text, analysis_text, period, ...)`**

**スライド構成**

| スライド種別 | 条件 | 内容 |
|------------|-----|------|
| テンプレートスライド | 常に生成 | プレースホルダー置換 |
| 商品別表スライド | slide_product_table=True | 四半期×商品 マトリクス |
| 地域別表スライド | slide_region_table=True | 四半期×地域 マトリクス |
| 担当者別表スライド | slide_rep_table=True | 四半期×担当者 (上位8) マトリクス |
| グラフスライド | slide_chart=True | 月別棒グラフ + 商品別グラフ |

**プレースホルダー置換**

| プレースホルダー | 置換内容 |
|--------------|---------|
| {{report_title}} | 報告書タイトル（期間含む） |
| {{report_date}} | 生成日（YYYY年MM月DD日） |
| {{summary_text}} | Writer AI 生成サマリー |
| {{analysis_text}} | Writer AI 生成分析 |

**表スライド設計**

- レイアウト: 16:9（12,192,000 × 6,858,000 EMU）
- カラーパレット:
  - ヘッダー背景: Navy `#0A1228`
  - ヘッダー文字: Gold `#D4941A`
  - 偶数行背景: Teal `#00A39A` (透過)
  - 奇数行背景: 白
- 金額単位: 万円（10,000 で除算・小数点以下丸め）

**グラフスライド設計**

- matplotlib で PNG 生成 → PPTX に埋め込み
- 左グラフ: 月別売上棒グラフ（`利益額` あれば利益率折れ線を追加）
- 右グラフ: 商品別棒グラフまたは円グラフ（上位 6 商品）
- カラー: `["#0A1228", "#D4941A", "#00A39A", "#4472C4", "#ED7D31", "#A9D18E"]`

#### 1.4.4 `services/rag_store.py`

**ChromaDB 設定**

```python
client = chromadb.PersistentClient(path=CHROMA_DIR)
collection = client.get_or_create_collection(
    name="past_reports",
    metadata={"hnsw:space": "cosine"}
)
```

**`register_report(pptx_path, filename)`**

1. `extract_chunks_from_pptx(pptx_path)` でスライド単位のテキスト抽出
2. 既存 `file_id` のデータを削除（再登録対応）
3. 各チャンクを `nomic-embed-text` で 768 次元ベクター化
4. ChromaDB に保存（メタデータ: `{filename, file_id, chunk_idx}`）

**`file_id` 生成**: ファイル名の MD5 ハッシュ（16進数）

**`extract_chunks_from_pptx(pptx_path)`**

- スライドごとにテキスト結合
- `{{...}}` プレースホルダーをタグ除去
- 30 文字未満のチャンクは除外

**`search_context(query, n_results=5)`**

1. `query` を `nomic-embed-text` でベクター化
2. ChromaDB で `n_results` 件の類似検索
3. コサイン距離 < (1 - RAG_THRESHOLD) のチャンクのみ採用
4. 採用チャンクを改行で連結し、先頭 1,500 文字を返却
5. 採用チャンクが 0 件の場合: 空文字列を返却

---

#### 1.4.5 `services/history_store.py`

**`append_history(job_id, original_filename, output_path, analyst_model, writer_model)`**

1. `_lock` を取得
2. `history.json` を読み込み（なければ空リスト）
3. 新エントリを先頭に追加
4. エントリ数 > MAX_HISTORY (50) の場合、末尾を削除し該当 PPTX ファイルも削除
5. `history.json` に書き戻し

エントリ構造:
```json
{
  "job_id": "ab12cd34ef56",
  "created_at": "2026-03-29T12:34:56Z",
  "original_filename": "sales_2024Q1.xlsx",
  "output_path": "/abs/path/output/report_ab12cd34ef56.pptx",
  "analyst_model": "qwen2.5:3b",
  "writer_model": "qwen3:8b"
}
```

**`list_history(n=20)`**

- `output_path` を除いたエントリを最新 n 件返却
- ファイルが実際に存在するエントリのみ返却

### 2.2 進捗ポーリングロジック

```typescript
const poll = async () => {
  const res = await fetch('/api/progress');
  const data = await res.json();
  setStep(data.step);
  if (data.done) {
    // ダウンロード処理
    const blob = await fetch('/api/download').then(r => r.blob());
    setDownloadUrl(URL.createObjectURL(blob));
    setStatus('success');
    sendNotification('完了');
    setHistoryKey(k => k + 1);
  } else if (data.error) {
    setStatus('error');
    setErrorMsg(data.error);
    sendNotification('エラー');
  } else {
    setTimeout(poll, 2000);  // 2秒後に再ポーリング
  }
};
// 最大 20 分 (600 回 × 2 秒) でタイムアウト
```

## 3. Ollama AI 詳細設計

### 3.1 2モデル分離アーキテクチャ

```
売上データ (raw_summary)
       ↓
┌──────────────┐
│ Analyst AI   │  qwen2.5:3b (軽量・高速)
│  役割: JSON  │  → 数値データ解析に特化
│  構造化出力  │  → 約 2〜5 分
└──────┬───────┘
       ↓ 構造化 JSON + 元データ + RAG コンテキスト
┌──────────────┐
│  Writer AI   │  qwen3:8b (高品質)
│  役割: 日本語│  → 自然言語生成に特化
│  文章生成    │  → 約 5〜15 分
└──────┬───────┘
       ↓ SUMMARY / ANALYSIS テキスト
  PPTX 生成へ
```

### 3.2 RAG 処理詳細

**ベクター検索フロー**

```
query = data["raw_summary"]  (売上集計テキスト)
    ↓
nomic-embed-text でベクター化 (768次元)
    ↓
ChromaDB cosine 類似度検索 (n_results=5)
    ↓
距離フィルタ (distance < 1 - 0.3 = 0.7 を採用)
    ↓
テキスト連結 → 先頭 1,500 文字
    ↓
Writer プロンプトに "[参考: 過去報告書の記述]" として付加
```

**登録時のチャンク分割**

- 単位: 1スライド = 1チャンク
- 最小長: 30 文字（短すぎるチャンクは除外）
- プレースホルダー除去: `{{...}}` パターンを削除
